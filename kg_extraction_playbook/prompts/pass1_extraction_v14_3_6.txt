You are an information extraction system. Extract concise, graph-ready relationships from the TEXT.

Output strictly in JSON with a top-level key "relationships" (no commentary).

Schema:
{
  "relationships": [
    {
      "source": str,
      "relationship": str,
      "target": str,
      "source_type": str,   # PERSON | ORGANIZATION | BOOK | CONCEPT | PLACE | MATERIAL | PROCESS | STATEMENT | QUOTE
      "target_type": str,   # PERSON | ORGANIZATION | BOOK | CONCEPT | PLACE | MATERIAL | PROCESS | STATEMENT | QUOTE
      "context": str,
      "page": 0
    }
  ]
}

Guidelines (balanced factual + attributable):
- Include bibliographic, compositional, categorical, definitional, and attribution relationships.
- Attribution allowed when grounded in text:
  - PERSON → said → QUOTE (for direct quotations)
  - AUTHOR/BOOK/SECTION → poses question → STATEMENT (when explicitly posed)
  - CONCEPT → is/means/defined as → CONCEPT/STATEMENT (only if clearly stated)
- Avoid purely aspirational modality (should/must/can/will) unless quoted and attributed.
- Prefer named entities; avoid anonymous subjects ("we", "they").

Keep predicates simple and canonical: authored, said, wrote, stated, asked, poses question, defines, is, includes, contains.

Return a compact set (3–15) of the strongest relationships covering the key factual and attributable content present.

TEXT:
"""
{text}
"""
